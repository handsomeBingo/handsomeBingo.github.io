        这篇文章本来打算讲一个VUE-CLI例子，但是大家可以先看看这篇博客，强烈建议按照博客代码敲上一遍。这个例子图文并茂，是我在学vue-cli的时候见到的最好的教程。上链接 史上最全入坑教程。

        这篇文章说说组件开发，我理解的组件开发，我们习惯一开始就先钻进代码里，迫不及待的看代码。大神自是心领神会，对于一些事情根本不需要说出来，这些大神没有说出来的，正是阻碍我们前进的障碍。所谓组件开发，是一种“拆”的思想或者说是“拼”，是把一个页面，按照某种规则，一般按照功能、复用性，UI样式等规则拆分成若干部分，最不理想的拆还可以掐头去尾留中间。。。。这份工作首先是切图，按照一个完整的网页，把图切好，然后拆成一块一块的，然后由不同的人开发，这就是最通俗的讲法。和拼图或者积木一类的很像。但是拆法是仁者见仁智者见智的时候，从这里能看出水平。

        我第一次接触VUE-CLI这个脚手架的时候，眼前一亮，但是同时初次接触vue，这么多 *.vue的文件直接让我懵逼了。现在我走过来了，今天就可以说说这些玩意儿了。

        第一个让我困惑的问题是父子组件的问题；每一个.vue的文件都是一个组件，在组件中被import的相对于该.vue而言就是子组件（后面须注册）。在学习VUE初期，我对于父子组件一直感到很困惑，因为在VUE中，父子组件和Angular中的父子组件完全不同，Angular的作用域是和DOM平齐的，而父子组件也会因为DOM嵌套关系而天然形成，而VUE组件的父子关系与嵌套无关，而是与注册关系决定的，习惯了这一思路后，我着实是在这里困惑了好久，如果眼前的你也是这样的话，那就赶紧出来吧。

        下面代码是 point.vue 组件的代码（部分），其中point.vue相对于common-header和mask-layer是父组件，而common-header和mask-layer是子组件。


    <template lang="html">
        <common-header :isNotLogin="isNotLogin" :theActive="active"></common-header>
        <mask-layer ></mask-layer>
      </div>
    </template>


    <script>
      import CommonHeader from '../common/CommonHeader' //导入子组件
      import MaskLayer from '../common/MaskLayer' //导入子组件
      export default {
        name: 'pointOperate',
        data () {
          return {
            newMember: {
              mName: '',
              mNum: '',
              point: ''
            }
        },
        methods: {},
        components: {
          CommonHeader, //注册子组件
          MaskLayer //注册子组件
        }
      }
    </script>

        另一个困惑我很久的问题是如何在VUE-CLI中引入自己的静态资源文件，如bootstrap.min.css。写完这些基础内容后，我将分享一个完整的例子，而且，这个例子已经投入使用了（这个积分管理系统是给我女友的奶茶店用的积分管理），现在不多说，继续写基础内容。引入VUE-CLI后，脚手架将自动为我们打包应用，并将打包后的文件自动注入到index.html中，正由于这一操作，如果我们不按照正确的操作方式，我们自己引入的静态文件将失效。正确的做法是：在index.html同级新建一个static文件夹，将我们自己的静态文件放入其中，在index.html中使用相对路径引入即可。下面是index.html的代码：


在head标签中引入index.css和bootstrap.min.css等静态文件
    <!DOCTYPE html>
    <html>
      <head>
        <meta charset="utf-8">
        <title>admin</title>
        <meta name="viewport"
              content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
        <title>index首页</title>
        <link rel="stylesheet" href="./static/bootstrap/dist/css/bootstrap.min.css">
        <link rel="shotcut icon" href="./static/favicon.ico">
        <link rel="stylesheet" href="./static/index.css">
      </head>
      <body>
        <div id="app"></div>
        <!-- built files will be auto injected -->
      </body>
    </html>



目录结构
--|project
    --|build/
    --|config/
    --|dist/
    --|node_modules/
    --|src/
    --|static/
        --|bootstrap/ ==>bootstrap.min.css所在文件夹
            --|dis/
            --|fonts/
            --|...
        --|index.css
    --|.babelrc
    --|.editorconfig
    --|.gitignore
    --|.postcsssrc.js
    --|index.html
    --|package.json
    --|README.md

        第三个让我困惑的问题是关于数据驱动。在写这个系统的时候，我需要一个bootstrap的modal框，这个框用于确认、提示用，在bootstrap中用jquery操作点击事件来实现modal框的显隐，在vue虽然也可以操作dom，但是我最终选择了放弃这一做法，如果这样做了，那还算用框架么？我困惑了很久，其实现在想来特别简单，只需要设置modal框的v-show即可。想通了自然很简单，当时也是初学vue，根本不知道从何入手，初学vue容易只见树木不见森林，我们纠结语法和API，忽略了指导性的核心思想——数据驱动。我们想要实现某个功能时，我们思考的方向是改变某个（些）值（数据）来达到。
