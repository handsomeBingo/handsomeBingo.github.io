LESS入门教程

LESS入门教程

为什么学习LESS

　　说实话，起初我特别反感写LESS这种需要编译的东西，因此一直心怀抵触，但这过程中我没放弃学习这些技术，原因很简单，辩证来看，这种技术如果真像我厌恶的样子它怎么会流行？我厌恶，说明是我还没真正窥见她的美。当你讨厌的时候，你有数不清的理由，但是你爱上她，你发现根本没有理由。

　　我的leader就用LESS开发，也要求过我用，我一直反对，直到后来一些我未曾预见的问题找上门——官网改版，新老版共存，CSS类名经常发生冲突，让我异常崩溃；我的leader再次推荐LESS，我才正式的运用这项技术。我才发现LESS写出的css样式代码真的很美，分享这项技术我觉得不一定是重点，重点是心路历程。一项技术的诞生都是为了解决一些问题或者能够提升开发效率。

初学LESS我们需要掌握什么

　　首先，我们需要知道它就是css样式代码。但是初学，我们需要了解只有以下几点

  • 安装
  • 变量思想 【块级作用域（变量查找机制）】
  • 层级与嵌套 【后代则直接嵌套、&与伪类】
  • 函数化css【函数调用样式、@arguments、默认值、Mixin、继承样式、复用子级样式】

安装LESS

这里推荐使用NODEJS + LESS的学习方式；
1.安装nodejs
2.安装LESS npm install -g less

LESS变量

　　LESS的变量思想类似JS的变量（值类型变量）。LESS的变量只能是值类型，可以使色值、字体等等；
　　LESS声明变量使用@符号
　　LESS的变量和JS的变量很类似，存在作用域和作用域链机制。在LESS中，选择器后面的花括号即块级作用域（私有作用域），所以，未在选择器后的花括号内声明的变量都是全局变量，可以在任何的位置访问，在选择器后的花括号内声明的变量都是私有变量，只能在当前块级作用域中使用；
　　块级作用域是一个命名空间——场景-当我们拥有了大量的选择器的时候，特别是团队协同开发时，如何保证选择器之间重名问题：LESS采用了命名空间的方法来来避免重名问题； 

@width: 900px;//全局变量
@bgColor: blue;//全局变量
.contentHeart {
    @bgColor: #00b38a;
    width: @width;
    background:  @bgColor;//使用私有变量，在LESS中，变量查找也是采取就近原则，如果当前作用域中存在该变量，则直接取该作用域下的值，如果没有则向上查找
}
//编译之后的css样式
.contentHeart {
    width: 900px;
    background: #00b38a;
}

设置嵌套HTML结构样式

　　在LESS中如果设置被嵌套的样式，则直接在父级元素样式的花括号内继续写欲设置的子级元素的选择器样式，示例： 

//html
<div id="home" >
    <div id="top">top</div>
    <div id="center">
        <div id="left">left</div>
        <div id="right">right</div>
    </div>
</div>


//LESS样式，在LESS中，关于HTML嵌套，我们采用从外到内的选择器嵌套定义，要么采用给特定元素加class或id
#home {//父级
  color: blue;
  width: 600px;
  height: 500px;
  border: outset;
  #top {//子级
    border: outset;
    width: 90%;
  }
  #center {//子级
    border: outset;
    height: 300px;
    width: 90%;
    #left {//子级的子级
      border: outset;
      float: left;
      width: 40%
    }
    #right {//子级的子级
      border: outset;
      float: left;
      width: 40%;
    }
  }
}
//编译之后的css样式
#home {
  color: blue;
  width: 600px;
  height: 500px;
  border: outset;
}
#home #top {
  border: outset;
  width: 90%;
}
#home #center {
  border: outset;
  height: 300px;
  width: 90%;
}
#home #center #left {
  border: outset;
  float: left;
  width: 40%;
}
#home #center #right {
  border: outset;
  float: left;
  width: 40%;
}

如果你要复用div#home下的div#top的样式，只需要在其他选择器中这样写：


.copyStyle {
    #home>#top;
}

在嵌套中最值得一提的是伪类，在嵌套中 有 &时解析的是同一个元素或此元素的伪类；没有 & 时，解析的是后代元素；示例：


//设置a：hover样式
a {
  color: red;
  text-decoration: none;
  &:hover { //有 & 时解析的是同一个元素或此元素的伪类；没有 & 时，解析的是后代元素
    color: #000;
    text-decoration: underline;
  }
}
//编译后的样式
a {
  color: red;
  text-decoration: none;
}
a:hover {
  color: #000;
  text-decoration: underline;
}

在LESS中使用函数

LESS还有一个更为强大的功能——函数；这里我们不讨论LESS支持的表达式及色值的计算函数；只是类似JS函数的一种用法；
Mixins——Mixins其实是一种嵌套，它允许将一个类嵌入到另外一个类中使用，被嵌入的类也可以称作变量，简单的讲，Mixins其实是规则级别的复用。说通俗一些，就是直接把一个选择器直接放到新的块级作用域中即可； Mixins还有一种形式叫做ParametricMixins（混入参数），Less也支持这一特性 示例： 

//定义一个样式选择器选择器函数
.roundedCorners(@radius: 5px) {// 5px是一个默认值
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  border-radius: @radius;
}

//在另外的样式选择器中使用
#header {
  .roundedCorners;//不传参，则使用默认值5px；同时这也是一个Mixins
}
#footer {
  .roundedCorners(10px);//传参则使用实参10px；同时这也是带参数的Mixins
}

//编译后代码
#header {
  -webkit-border-radius: 5px;
  -moz-border-radius: 5px;
  border-radius: 5px;
}
#footer {
  -webkit-border-radius: 10px;
  -moz-border-radius: 10px;
  border-radius: 10px;
}

@arguments——在LESS的函数中，同样也有一个类似JS函数中arguments的东东，就是@arguments，他同样包含着函数所有的参数，与JS的arguments不同的是，@arguments是值的传递；示例：


//定义一个函数，这个函数参数众多
.boxShadow(@x: 0, @y: 0, @blur: 1px, @color: #000) {
  -webkit-box-shadow: @arguments;
  -moz-box-shadow: @arguments;
  box-shadow: @arguments;
}

//使用
#header {
  .boxShadow(2px, 2px, 3px, #f36);
}

//编译后的样式代码
#header {
  -webkit-box-shadow: 2px 2px 3px #f36;
  -moz-box-shadow: 2px 2px 3px #f36;
  box-shadow: 2px 2px 3px #f36;
}
