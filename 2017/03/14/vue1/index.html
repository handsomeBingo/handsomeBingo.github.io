vue从入门到掉进坑里（一）

        学习vue好一阵子了，是从一个爱炫耀的同事开始的。说到vue，我只想说：太坑了；无论文档还是某些人博客，都写的很晦涩。这让我很不解，我觉得这和打码是一样的。就不能写的通俗一些？说喜欢分享来着，结果分享的东西晦涩的不得了。我称这些博客和文档为：打码的文章。说到这个我又想起我一个同事问我的一个问题：什么样的人算大牛？我当时想了半天也说不清，我想coding能力强的人大抵就是大牛了。他的答案让我很意外，原话引用“写出来谁都看不懂的，不算大牛；写出来谁都看得懂的，也不是大牛；只有那些你看了能看懂，但是看完就拍大腿：我怎么就没想到呢？这些才是大牛。”我以此自砺，也送给那些给文章打码的人。

vue基础

        如果你有angular 1.xx的基础就足够了。但是作为一个还在坑边上歇息的人告诉你，别拿着angular的思想去衡量vue的问题，尤其是 2.0以后的版本。基础不多说直接引用官网的教程：

hello world

使用Vue的过程即定义MVVM各个组成部分的过程。
* 1.定义View
* 2.定义Model
* 3.创建一个Vue实例或’ViewModel’，它用于连接View和Model；在创建Vue实例时，需要传入一个选项对象，选项对象可以包含数据、挂载元素、方法、模拟生命周期钩子等等。
* 示例讲解：
* 在这个示例中，选项对象的el属性指向View，el:’#app’，表示将Vue示例挂载到 <div id = ‘app’></div>元素上；
* data属性：指向Model，data:exampleData表示我们的Model是exampleData对象；
* Vue.js有多种数据绑定的语法，最基础的形式是文本差值，使用一对大括号语法,mustache风格绑定；

双向动态绑定

绑定是双向绑定的，不同于angular 1.xx数据是挂载在作用域$scope上，vue的开发将数据都集成在了vue实例化的参数对象的data属性中，这一点无比的方便，我们推崇vue，但是拒绝装B，尤其是那些以vue牛逼，自己也牛逼起来的人。
